# Plan: Обучающие квесты (Пилот)

## Цель

Два квеста для пилотного запуска, обучающих вайбкодеров архитектурным концептам через геймплей.

**Методология:** "Сначала опыт — потом термин". Игрок решает задачи, которые *являются* концептом, но не знает термина. Формальное название появляется только в финале.

**Целевая аудитория:** Вайбкодеры (1000+), которые строят проекты через AI без понимания архитектуры → вечные переделки → перерасход токенов/энергии.

---

## Параметры

| Параметр | Решение |
|----------|---------|
| Квесты | 2 (полиморфизм + модульная архитектура) |
| Формат | Выбор из 2-3 вариантов |
| Последствия | Мягкие (одна линия + штрафные шаги) |
| Ветвление | Нет разных веток, есть "долг" от плохих решений |
| Тон | Юмористический sci-fi |
| Термин | Только в финале |
| Один квест | Один концепт |

## Квест А: Полиморфизм ("Зоопарк дронов")

### Концепт

Игрок управляет дронами через диспетчерскую. Сначала видит подход с `if/else` на каждый тип дрона, потом через выбор приходит к единому интерфейсу `.execute()`. Термин "полиморфизм" появляется только на финальном экране.

### Структура (5 локаций)

#### Локация 1 — Диспетчерская (spawn)

Контроллер дронов сломан. 3 дрона стоят без дела: разведчик, грузовой, ремонтный. Нужно починить управление.

#### Локация 2 — Первый вызов

Сигнал: астероид на пути. Нужно отправить дрона на разведку. Система показывает код с `if/else`.

**Выбор (InteractiveObject с 2 actions):**
- **A)** Оставить как есть, работает же → *устанавливает флаг `chose_ifelse`*
- **B)** Переписать, чтобы у каждого дрона был метод `.execute()` → *устанавливает флаг `chose_polymorphic`*

Оба варианта проходят. Разница видна позже.

#### Локация 3 — Второй вызов, усложнение

Добавили 2 новых дрона: медицинский и боевой. Система требует обновить код.

**Если `chose_ifelse`:**
Код уже на 15 строк. Выбор:
- **A)** Дописать ещё 2 ветки `else if` → *эффект: -10 Focus, lintSay "система перегружена"*
- **B)** Переделать на `.execute()` → *код сокращается, lintSay "стабильно"*

**Если `chose_polymorphic`:**
Просто добавляешь 2 новых дрона. Система: "2 строки добавлены, 0 изменено". Проходишь без штрафа.

#### Локация 4 — Кризис

10 новых типов дронов от союзников. Нужно интегрировать ВСЕ.

**Путь if/else:** Стена из 30+ строк. Штраф HP/Focus. Лишняя локация "исправление багов".

**Путь .execute():** Одна строка `drones.forEach(d => d.execute(mission))`. Быстро проходит.

#### Локация 5 — Терминал откровения (финал)

Бортовой AI выводит сравнение двух подходов:

```
Путь A: 47 строк, 12 if/else, 3 бага при добавлении
Путь B: 11 строк, 0 if/else, 0 багов

Один интерфейс (.execute), разное поведение у каждого дрона.
Это называется ПОЛИМОРФИЗМ.
"Poly" = много, "morph" = форма. Много форм — одна команда.

Токены AI: путь A = 8000, путь B = 1200. Экономия 85%.
```

`completionCondition: { hasFlag: "quest_complete" }`

## Квест Б: Модульная архитектура ("Вирус на станции")

### Концепт

Станция работает на одном файле `station.js` (2000 строк). Игрок чинит баги и сталкивается с каскадными сбоями. Через выборы приходит к модульности с интерфейсами. Термин "модульная архитектура" — в финале.

### Структура (5 локаций)

#### Локация 1 — Серверная (spawn)

Станция на одном файле. Баг-репорт: двери не открываются.

**Выбор (3 actions):**
- **A)** Починить баг прямо в `station.js` → *флаг `chose_monolith`, эффект: -10 HP (сломал освещение)*
- **B)** Вынести систему дверей в `doors.js` с интерфейсом → *флаг `chose_modular`*
- **C)** Вынести в файл, но без интерфейса (глобальные переменные) → *флаг `chose_no_interface`*

#### Локация 2 — Каскадный сбой

**Путь A (монолит):** Освещение → навигация → гравитация. Цепная реакция. -20 HP, -15 Focus. lintSay: "Изменение в строке 847 затронуло строки 12, 340, 1205, 1899".

**Путь C (файлы без интерфейса):** Частичный сбой. -10 HP. lintSay: "Глобальные переменные — ненадёжная связь".

**Путь B (модули с интерфейсом):** Всё стабильно. lintSay: "Модуль doors.js изменён. Остальные не затронуты."

Новая задача: обновить систему кислорода.

#### Локация 3 — Новый модуль

Выбор подхода к обновлению кислорода:

- **A)** Дописать в существующий файл → *файл растёт, конфликты*
- **B)** `oxygen.js` с интерфейсом `{ getLevel(), setLevel(), alert() }` → *чисто*
- **C)** Отдельный файл с глобальными переменными → *ловушка*

#### Локация 4 — Вирусная атака

Вирус распространяется по связям между системами.

**Монолит:** 100% заражение за 10 секунд. Экстра-локация "экстренное восстановление". -30 HP.

**Файлы без интерфейсов:** 60% заражение через глобальные переменные. -15 HP.

**Модули с интерфейсами:** 10% заражение. Игрок изолирует модуль → заменяет на чистую копию. Минимальный штраф.

#### Локация 5 — Панель мониторинга (финал)

Финальный экран со сравнением:

```
                    Монолит    Файлы без    Модули с
                               контракта   интерфейсом
Зона поражения      100%        60%          10%
Время починки       2 часа      45 мин       5 мин
Строк изменено      340         89           12
Побочные эффекты    17          5            0

Независимые части с чёткими границами и контрактами.
Это называется МОДУЛЬНАЯ АРХИТЕКТУРА.

Токены AI: монолит = 12000, модули = 2400. Экономия 80%.
```

`completionCondition: { hasFlag: "quest_complete" }`

## Технические решения

### Реализация "выбора" в текущем движке

Каждый "вопрос" — это `InteractiveObject` с 2-3 `ObjectAction`:

```json
{
  "id": "choice_1",
  "name": "Панель управления",
  "description": "Как починить код?",
  "actions": [
    {
      "id": "option_a",
      "text": "Вариант A: оставить как есть",
      "effects": [
        { "type": "setFlag", "target": "chose_ifelse" },
        { "type": "setFlag", "target": "choice_1_made" }
      ],
      "requires": { "notHasFlag": "choice_1_made" },
      "successMessage": "Ты выбрал путь if/else..."
    },
    {
      "id": "option_b",
      "text": "Вариант B: единый интерфейс",
      "effects": [
        { "type": "setFlag", "target": "chose_polymorphic" },
        { "type": "setFlag", "target": "choice_1_made" }
      ],
      "requires": { "notHasFlag": "choice_1_made" },
      "successMessage": "Ты выбрал единый интерфейс..."
    }
  ]
}
```

Флаг `choice_1_made` скрывает все варианты после первого выбора.

### Мягкие последствия через флаги

```
chose_ifelse → доп. локация "исправление багов" (requires: hasFlag)
chose_polymorphic → пропускает доп. локации
```

Штрафы: `modifyHp`, `modifyFocus`. Не блокируют, но ухудшают состояние.

### Файловая структура

```
src/data/levels/
├── level-1-hello-world.ts          (существует)
├── level-reference.ts              (существует)
├── level-polymorphism.ts           (новый — Квест А)
└── level-modular-architecture.ts   (новый — Квест Б)
```

Оба уровня регистрируются в `src/engine/levelRegistry.ts`.

## Дизайн-принципы

1. **Последствия видны** — штрафы HP/Focus, доп. локации при плохих выборах
2. **Нет Game Over** — плохой путь сложнее, но проходим
3. **Термин в финале** — весь квест интуитивно, название только на последнем экране
4. **Метрика токенов** — финальный экран показывает экономию при правильной архитектуре
5. **Юмор L.I.N.T.** — саркастичный AI комментирует каждый выбор

## Порядок реализации
